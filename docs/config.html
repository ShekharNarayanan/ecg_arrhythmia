config.yaml

Central configuration file for the arrhythmia beat classification pipeline. Loaded at runtime by train.py, eval.py, and shared utilities via ml_utils.py. All paths, signal processing parameters, feature extraction settings, training hyperparameters, and experiment definitions live here so that no magic numbers appear in code.

# section paths

Filesystem and MLflow tracking locations used across the pipeline.

KeyDefaultDescription
raw_data_path 'data/raw' Directory containing the raw MIT-BIH .dat / .hea / .atr recording files.
mlflow_exported 'data/mlflow_exported' Output directory for any manually exported MLflow artifacts (e.g. CSV run summaries).
mlflow_tracking_uri 'mlruns' MLflow backend store URI. Set to 'sqlite:///mlflow.db' for cross-OS compatibility when moving the project between Mac and Windows.
mlflow_experiment_name 'arrhythmia_beat_classification' Logical experiment group name shown in the MLflow UI. All runs from the ablation are logged under this name.
# section signal_processing

Bandpass and notch filter parameters applied during preprocessing (preprocess.py) and beat segmentation (beats.py).

KeyDefaultDescription
general_bandpass [0.5, 30.0] Butterworth bandpass cutoffs (Hz) applied to the full ECG recording. Removes baseline wander below 0.5 Hz and high-frequency noise above 30 Hz.
notch 50 Notch filter frequency (Hz) to suppress power-line interference. Use 60 for North American recordings.
beats_bandpass [1, 15] Tighter bandpass applied to individual beat segments before waveform feature extraction. Emphasises QRS content and attenuates T-wave and baseline drift.
peak_calibration 80 Half-window in milliseconds used to align algorithmically detected R-peaks with the MIT-BIH annotation file. The detected peak is snapped to the annotated location if it falls within this window.
# section feature_extraction

Controls how waveform segments, RR-interval features, QRS morphology features, and wavelet coefficients are computed in extract_features.py.

KeyDefaultDescription
wfm_extraction_window [-200, 400] Time window in milliseconds relative to the R-peak used to cut out a beat waveform. -200 captures the P-wave onset; 400 covers the T-wave tail. Converted to samples internally using the recording's sampling frequency.
local_rr_mean_beat_window 5 Number of preceding beats used to compute the local RR mean for a given beat. This mean is used to derive deviation_rr (current RR minus local mean) as one of the interval-related features.
qrs_extraction_window [-200, 60] Search window in milliseconds relative to the R-peak for locating the Q trough (left side) and S trough (right side). Q is searched in the 200 ms before the R-peak; S is searched in the 60 ms after.
wavelet_decomp_level 4 Number of decomposition levels for the discrete wavelet transform (DWT) using PyWavelets (db2 wavelet). Detail coefficients from all levels are concatenated into a single feature vector. Note: in ablation experiments, wavelet features were found to hurt performance, likely due to noise introduced by concatenating across levels.
rr_irregularity_window 64 Number of beats on each side of the current beat used to compute the RR coefficient of variation (rr_irreg). A larger window produces a smoother irregularity estimate. Controlled per-experiment via include_rr_irregularity.
# section model_training

Global training parameters shared across all experiments. Individual experiments can override classifier-specific hyperparameters via their classifier_params block.

KeyDefaultDescription
max_iter 2000 Maximum solver iterations for logistic regression. Passed to LogisticRegression(max_iter=...). Has no effect on XGBoost experiments.
cv_split_param 5 Number of folds for cross-validation (e.g. StratifiedKFold or GroupKFold when patient-wise splitting is used). Also controls the cv argument in RandomizedSearchCV.
n_iter_random_search 30 Number of parameter combinations sampled during RandomizedSearchCV hyperparameter tuning. Only relevant for experiments that include a search_CV_params block.
# section label_filtering

Determines which beat classes are retained from the MIT-BIH annotations before feature extraction and model training.

KeyDefaultDescription
keep_labels ['N', 'A', 'V'] List of AAMI beat-type symbols to retain. N = Normal, A = Atrial premature beat, V = Ventricular premature beat. All other annotation types are discarded. Class imbalance is handled at training time via class_weight="balanced" rather than oversampling.
# section experiments

Defines the full ablation suite. Each top-level key (e.g. exp01_waves_logreg) is a named experiment that train.py iterates over. The config drives a feature group ablation across two classifiers — logistic regression and XGBoost — following the progression:

waves only → waves + RR → waves + RR + QRS → waves + RR + QRS + wavelet

Each experiment entry supports the following keys:

KeyTypeDescription
run_name str Label shown in the MLflow UI for this run. Typically matches the experiment key name.
features list[str] Ordered list of feature groups to include. Valid values: 'waves', 'interval_related', 'QRS', 'wavelet'. Note: 'waves' is required when 'QRS' or 'wavelet' are included, as they depend on the waveform extraction window.
classifier str Classifier to use. One of 'logreg' (logistic regression via scikit-learn) or 'xgboost' (gradient boosting via XGBoost). The pipeline is built inside ml_utils.build_pipeline().
oversample bool Whether to apply SMOTE oversampling. Kept in the schema for reference; set to False across all ablation runs — SMOTE was found to degrade performance and was replaced by class_weight="balanced".
include_rr_irregularity bool Whether to append the rr_irreg coefficient-of-variation feature to the interval-related feature group. Only has an effect when 'interval_related' is listed in features. In XGBoost ablation experiments, toggling this flag produced identical results — likely because rr_irreg is redundant with delta_rr and deviation_rr.
classifier_params dict Hyperparameters passed directly to build_pipeline(). For logreg: max_iter, solver, verbose. For xgboost: n_estimators, colsample_bylevel, tree_method, device.
search_CV_params dict (optional) Parameter grid for RandomizedSearchCV. Keys must be prefixed with clf__ to target the classifier step in the pipeline (e.g. clf__n_estimators). Omit this key to run a fixed-parameter experiment.
description str Free-text note logged as an MLflow tag. Used to annotate what each experiment tests in the ablation story.
